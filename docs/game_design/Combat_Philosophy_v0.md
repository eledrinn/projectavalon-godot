# Combat Philosophy v0

## Status
LOCKED — Targeting model and combat pacing defined.

## Targeting Model (LOCKED)
- **Soft-target hybrid** with tab-target available for auto-attacks
- **Classic MMO pacing:** Swing-timer auto-attacks + active abilities learned through progression
- **Selective ballistic simulation:** Some projectiles use physics with realistic trajectories (Ranger shots, Apogist gravity hurls)
- **Not action-combat:** No dodge-spam, no animation-cancel dependency
- **Readable and rhythmic:** Combat feels slower, deliberate, tactical — like classic MMOs

## Death & Respawn Philosophy (LOCKED)
### Hardcore Mode (Default for New Characters)
- Character starts with "Oathbound" buff (one life, better rewards)
- Slightly higher quality gear drops while buff is active
- First death removes buff, character continues in Normal mode
- Hardcore characters who die may optionally delete or convert to Normal

### Normal Mode (Post-First-Death)
- **Corpse loot system:** All gear remains on body at death location
- **Shrine respawn:** Return to last bound shrine as spirit/echo form
- **Corpse run:** Must travel back to body in spirit form to recover gear
- **Spirit form:** Can traverse terrain quickly but cannot fight or interact
- **Cultural flavor:**
  - **Natives:** "The Basin Remembers" — respawn at sacred wetland shrine
  - **Arrivals:** "Beacon Recall" — emergency return to expedition anchor
  - **Severed:** "Pressure Lock" — consciousness reset to nearest stable point

## Core Rule
Combat should be:
- readable
- expressive
- terrain-aware
- class-distinct
- group-friendly
- immersive without becoming chaotic

The game should not be fully tab-target.
The game should not be fully action-combat.
The sweet spot is hybrid.

## What Hybrid Means Here
Hybrid means:
- ability-driven combat
- MMO readability
- strong class roles and ability loops
- some physicality in movement, projectile travel, terrain use, and timing
- enough immediacy to feel alive, but enough structure to remain legible in group play

## Combat Goals
1. Class identity is obvious quickly
2. Elevation and terrain matter
3. Some projectile behavior can feel physical and satisfying
4. Combat should scale to multiplayer without becoming unreadable
5. Positioning matters
6. Timing matters
7. The game should still feel like an RPG, not a pure action brawler

## What to Avoid
- full twitch dependency
- animation-locked action combat across everything
- unreadable particle chaos
- fully passive tab-target autopilot feeling
- complete realism if it harms clarity or fairness
- pure spreadsheet combat with no sense of place

## Physical / Ballistic Philosophy
Selective ballistic or physical simulation is allowed where it adds:
- flavor
- class fantasy
- terrain interaction
- moment-to-moment satisfaction

### Good candidates
- Apogist stone/gravity hurls
- some Ranger shots
- knockbacks or falls near cliffs
- Forceshaper wind displacement or storm arcs
- gravity or collapse effects

### Important limit
Not every attack should become a true simulation problem.
Readability and control matter more than realism.

## Terrain and Elevation
Terrain should matter in combat.

Examples:
- cliff edges create risk and opportunity
- line of sight matters
- elevation changes projectile feel or sightlines
- environmental hazards like gas pockets or heat vents can matter
- movement space should shape encounter tone

The world should not just be backdrop.

## Class Identity Through Combat
Each class should feel different in more than lore.

### Current class combat signatures
- Apogist -> constellation finisher / stars vs gravity-collapse split
- Tephrate -> pressure / vent / faults
- Bonder -> instinct / call-and-answer / aspect visions
- Forceshaper -> confluence / elemental sequencing
- Armsman -> openings
- Ranger -> read the line

These mechanics should matter early.

## Role Philosophy
The game currently implies role expression, but should avoid flattening all classes into sameness.

Target direction:
- clear class strengths
- role variety
- room for party synergy
- no complete homogenization

Open question:
how strictly to enforce “holy trinity” expectations vs softer role blending

## Group Readability
Combat should remain readable in group play.

That means:
- silhouettes matter
- telegraphs matter
- effect language matters
- each class’s core rhythm should be understandable
- support/healing/control effects must be legible without overloading the screen

## Movement
Open question:
how active and free movement should be

Tentative direction:
- mobility matters
- movement should feel responsive
- but not every fight should demand full action-MMO mobility expression

## Early Vertical Slice Requirement
The first combat prototype should prove:
- movement feels good
- one class mechanic feels good
- one enemy encounter feels good
- height/terrain matters at least a little
- the game feels like its own thing

## Open Questions
- dodge/roll/stamina philosophy (low priority — classic MMO feel suggests limited dodge)
- hit-scan vs projectile balance for specific abilities
- PvP assumptions (long-term consideration)
- party role strictness (holy trinity vs soft blending)
- animation priority and cancel philosophy (classic feel suggests minimal canceling)

## Resolved Questions
- **Targeting model:** LOCKED — Soft-target hybrid with tab-target auto-attacks
- **Death/respawn:** LOCKED — Hardcore default → Normal mode with corpse run
- **Combat pacing:** LOCKED — Classic MMO swing-timer feel, not action-combat

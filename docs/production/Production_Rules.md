# Production Rules v0

## Project Identity
We are building a stylized online RPG with classic-MMO soul.

Core pillars:
- Seamless overworld
- No loading screens except instanced content like dungeons, raids, and special scenarios
- Globe-compatible architecture from day one
- Future global circumnavigation should remain possible
- One frontier vertical slice first
- Strong class identity
- Exploration, place, and travel matter
- Modern tools and AI-assisted workflow, but classic MMO spirit

## Hard Pillars
These are non-negotiable.

1. No overworld loading screens
2. Planetary world in lore and architecture
3. Globe-compatible foundation, even if the first implementation is one frontier slice
4. Future circumnavigation must remain possible
5. Starter vertical slice first
6. Procedural or semi-procedural terrain supports macro landforms
7. Hand-authored landmarks and gameplay spaces provide memorable content
8. Stylized readable visuals
9. Classic MMO pacing, world scale, and exploration
10. Canon lives in docs, not in chat memory

## Scope Rules
1. We are not building a full MMO at once.
2. We are building a 1–10 frontier vertical slice first.
3. Every feature must justify itself for either:
   - the 1–10 vertical slice, or
   - the long-term technical/world foundation
4. No agent may expand scope casually.
5. “Cool” alone is not enough justification.

## World Rules
1. The world must feel alien but believable.
2. Biomes must flow logically.
3. Earth-readable ecology, alien details.
4. No simplistic morality framing.
5. No “natives good / arrivals bad” shorthand.
6. No sci-fi drift for Arrivals.
7. No generic steampunk drift for Severed.
8. Native culture is wetland-rooted and later highland-adapted.
9. Arrivals are sacred-classical, ordered, geometric, formal, and frontier-rustic.
10. Severed are survival-engineered descendants of the same roots.

## Gameplay Rules
1. Combat is hybrid.
2. Combat is not fully tab-target.
3. Combat is not fully action.
4. Selective ballistic or physical simulation is allowed where it improves fantasy and readability.
5. Terrain and elevation should matter.
6. Progression should emphasize identity, sidegrades, discovery, and class/world integration over pure stat inflation.
7. Friction should create meaning, not tedium.
8. Travel should feel real, not padded.
9. Every class needs one unforgettable signature mechanic.

## Production Rules
1. Canon lives in docs, not chat memory.
2. No implementation before a design brief exists.
3. No design brief before the relevant source-of-truth page exists.
4. Graybox before polish.
5. Procedural terrain for macro shape; handmade design for memorable spaces.
6. Contradictions must be surfaced immediately.
7. Agents may propose ideas within scope, but may not silently canonize them.
8. Every proposal must be labeled:
   - Locked
   - Tentative
   - Proposed
   - Open Question

## Long-Term Foundation Rules
1. The project base must stay maintainable for a small AI-assisted team.
2. The project must stay modular.
3. The project must stay understandable to a non-expert director.
4. Globe compatibility is required in architecture, not in immediate full implementation.
5. The overworld must remain seamless even as the world expands.

## Immediate Priority
The immediate priority is a solid design and technical foundation for:
- world bible
- class framework
- seamless overworld assumptions
- starter continent slice
- Native 1–5 starting experience
